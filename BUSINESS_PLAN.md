## 一、核心问题与解决方案

### 1.1 我们解决什么问题？

**DAO治理的现状是一团糟：**
- **碎片化**：1000+ DAO分布在50+条链，用户需要在Snapshot、Tally、Discourse、Discord之间来回切换
- **低参与率**：平均投票参与率仅3-10%，Uniswap治理投票率经常低于5%
- **信息过载**：一个提案动辄几千字，普通用户根本不知道该怎么投
- **没有即时激励**：投票是义务劳动，除了"对协议有好处"外，用户得不到任何即时回报

### 1.2 我们的解决方案

VoteNow = **治理聚合器 + AI分析师 + 激励层**

**核心功能：**
1. **一站式投票**：20+主流DAO的提案全部聚合在一个界面，一键投票（通过Snapshot签名）
2. **AI分析师（Governance Agent）**：DeepSeek驱动的专属分析师，用2句话告诉你提案是什么、风险多大、该怎么投
3. **投票即赚**：每次投票获得40-100积分，可兑换USDC、NFT、gas券

## 三、数据来源：100%真实，直连链上

### 3.1 数据从哪里来？

**不是爬虫，是官方API直连：**

| 数据源 | 获取方式 | 实时性 | 可靠性 |
|--------|---------|--------|--------|
| **提案数据** | Snapshot GraphQL API | 实时 | Snapshot是行业标准，Uniswap/Aave等都用它 |
| **投票权力** | 链上RPC调用（Alchemy/Infura） | 实时 | 读取用户token余额，决定投票权重 |
| **投票提交** | EIP-712签名 + Snapshot Hub | 实时 | 签名直接提交到Snapshot官方节点 |
| **用户积分** | 自建数据库（PostgreSQL） | 实时 | 投票成功后即时发放 |

### 3.2 为什么大家能在这里Vote？

**我们不是"替代"Snapshot，而是"增强"Snapshot：**

1. **Snapshot是基础设施**：所有投票数据存储在Snapshot Hub
2. **VoteNow是客户端**：我们提供更好用的界面 + AI分析 + 激励层
3. **投票流程**：
   - 用户在VoteNow选择投票选项
   - 生成EIP-712结构化消息
   - 用户用MetaMask签名（off-chain，无gas费）
   - 签名提交到Snapshot Hub（hub.snapshot.org）
   - Snapshot验证签名并记录投票
   - 投票结果在Snapshot官方页面可见

**换句话说**：在VoteNow投票 = 在Snapshot投票，只是体验更好。

### 3.3 Snapshot vs 链上治理：科普澄清

**这个问题很常见，甚至很多Web3从业者也混淆：**

#### 两种DAO治理机制

| 维度 | Snapshot（链下签名） | 链上治理（On-Chain） |
|------|---------------------|---------------------|
| **提案是什么** | JSON数据（标题、描述、选项） | 智能合约调用代码 |
| **存储位置** | Snapshot Hub服务器 | 区块链上（不可篡改） |
| **投票成本** | 0 gas（免费） | $5-50 gas/次 |
| **执行方式** | 多签钱包人工执行 | 合约自动执行 |
| **使用比例** | **80%的DAO** | 大型协议（Aave、Compound） |
| **示例** | Uniswap、Arbitrum、ENS | MakerDAO、Compound |

#### 为什么我们先做Snapshot？

**市场原因**：
- **80%的DAO治理**使用Snapshot（链下签名）
- 用户习惯了免费投票，不愿意付gas费
- 大多数提案不需要立即执行（社区信号先行）

**技术原因**：
- Snapshot API成熟、稳定、文档完善
- 无需处理复杂的链上合约交互
- 可以快速验证产品-market fit

#### 路线图：从Snapshot到全链上

```
Phase 1（当前）: Snapshot only
    ↓ 验证需求，积累用户
Phase 2: Hybrid（Snapshot + 链上读取）
    ↓ 展示链上治理数据
Phase 3: Full on-chain（Aave Governor等）
    ↓ 真正的协议级治理基础设施
```

#### 安全扫描对两种都重要

**Snapshot的风险**：**文字游戏**
```
提案写："优化协议参数，提升用户体验"
实际：人工执行时，多签把钱转走
```
**我们的AI检测**：标记大额资金转移、验证目标地址

**链上的风险**：**代码漏洞**
```
提案代码：upgradeTo(恶意合约)
结果：自动执行，协议被完全控制
```
**我们的AI检测**：分析合约调用，检查权限变更

**结论**：无论哪种治理方式，用户都需要AI安全分析。
